How It Works
In a full mesh, each SSR node forms one peering relationship with every other SSR node. For each peering relationship, one BGP session is formed through the overlay. That overlay can include one or more peer paths.

The number of peer paths in a peering relationship depends on WAN interfaces participating in the same overlay on both nodes. This calculator computes peer paths per overlay and totals them:
Total peer paths per node = (nodes - 1) x sum((WAN interfaces per overlay)^2)

Select a device type, number of nodes, overlays, and WAN interfaces per overlay to see warning and maximum mesh size limits.
